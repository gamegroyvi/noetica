# Noetica MVP-1 — отчёт о тестировании

**Как тестировал:** запустил debug-сборку Linux-десктопа после полного сноса локальной БД и SharedPreferences, прошёл основной end-to-end сценарий через GUI с записью экрана.

## Эскалации

Нет. Никаких блокеров и расхождений со спецификацией не нашёл.

## Результаты

- **PASS** — Чистый запуск открывает онбординг с пресетом 5 осей (Тело/Ум/Дело/Связи/Душа), кнопка «Создать пентаграмму» активна.
- **PASS** — После онбординга открывается «Лента», состояние пустое («Лента пуста»).
- **PASS** — `+` открывает шит «Новая запись» с переключателем Заметка/Задача и чипами осей с символами.
- **PASS** — Переключение в «Задача» добавляет ряды «Без дедлайна» и «XP при выполнении».
- **PASS** — Сохранение задачи закрывает шит и сразу появляется карточка в ленте с бейджем «задача», чипом «Тело» и «+80 XP».
- **PASS** — На вкладке «Задачи» задача показана с чекбоксом; нажатие чекбокса даёт зачёркивание заголовка.
- **PASS** — На вкладке «Я» пентаграмма растянута по верхней вершине (Тело), значение в списке `Тело = 40` (формула `80 × (1 - age/30д) / 200 × 100 ≈ 40` сразу после выполнения), остальные оси `0`. Стрик в углу: `1 д.`.
- **PASS** — Чёрно-белая палитра без градиентов соблюдена на всех экранах.

## Что не покрыл

- Дата/время дедлайна (`Без дедлайна` → пикер): кнопка работает, но пройти полный сценарий с дедлайном в этом тесте я не стал — не входит в основной XP-loop.
- Разрывы времени между записями (`+ N часов/дней`): требуют ≥2 записей с разными временами; визуальная фича, не часть scoring-loop.
- Heatmap, «год назад», капсулы, голос — не входят в MVP-1.
- Юнит-тесты и CI не запускал — на репо GitHub Actions не настроен; код-ревью не делал, только runtime.
