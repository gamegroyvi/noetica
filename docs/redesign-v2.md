# Noetica — план переработки v2

Документ-предложение, не код. Перед тем как рубить — обсуждаем и кромсаем здесь.

## Что не работает сейчас (по фидбэку)

1. **«Достиг 100 — что дальше»** — ось упирается в потолок и теряет смысл.
2. **Нейронка не помнит, что пользователь уже сделал** — генерит задачи в вакууме.
3. **Нет отчёта по выполнению** — задача закрылась = информации ноль.
4. **Просто «генератор задач», а не роадмап** — нет ощущения движения по плану.
5. **Все задачи в куче** — нет фильтров, сортировки, поиска.
6. **«Пентаграмма» — статика и не масштабируется** — название не работает при 3 или 9 осях, нет интерактива.
7. **Связь задача↔ось произвольная** — 40 XP за задачу, 2 XP в ось — почему 2, почему 40?
8. **Нет базы знаний о пользователе** — каждая генерация начинает с нуля.
9. **Сжигаются токены LLM** — без оптимизации это станет дорого.

## Предлагаемая архитектура

### 1. Прогрессия осей: уровни и витки

Вместо одной шкалы 0..100:

- Каждая ось имеет **уровень** (`level`, начинается с 1) и `progress` 0..100 в пределах уровня.
- При достижении 100 — `levelUp`: уровень ++, progress сбрасывается, **появляется анимация прорастания** (см. п.6).
- Сложность задач калибруется по уровню оси, не по «уровню в опроснике» (тот используется только для старта).
- Уровень оси визуально — концентрические кольца на «графе осей»: уровень 1 — внутренний многоугольник, уровень 5 — пятый виток снаружи.

### 2. Прямая связь задача ↔ ось ↔ XP (детерминированно)

LLM возвращает не `axisIds: [a, b]`, а:

```json
{
  "title": "...",
  "estimatedMinutes": 60,
  "difficulty": 1..5,
  "axisWeights": { "axis_id_1": 0.7, "axis_id_2": 0.3 }
}
```

`axisWeights` — суммируется в 1.0. Из этого считается:

```
basePoints = estimatedMinutes * difficulty / 10   // или подобная формула
// Каждая ось получает basePoints * weight
axisPoints[a] = basePoints * 0.7
axisPoints[b] = basePoints * 0.3
```

Никаких произвольных «40/2/2». Всё считается на клиенте, LLM не возвращает числа XP.

### 3. Отчёт по выполнению задачи (Task Reflection)

При отметке задачи как done — **необязательное модальное окно**:

- «Как прошло?» (`status`: easy/normal/hard/blocked)
- «Что получилось?» (текст, опционально)
- «Что не получилось / что мешало?» (текст, опционально)
- «Сколько потратил времени?» (опционально, default = `estimatedMinutes`)

Эти данные:
- Сохраняются в `TaskReflection` (новая таблица), привязаны к `Entry`.
- **Корректируют XP** — если `status=hard` и время > 1.5x estimated, XP +20%. Если `easy` и время < 0.5x — XP −20%.
- **Идут в персональную базу знаний** (см. п.7) для контекста при генерации новых задач.

Если пользователь скипает окно — задача просто закрывается, базовый XP начисляется без бонусов.

### 4. Чат с нейронкой (Coach Mode)

Заменяет однократный опросник на **диалоговый онбординг + еженедельную рефлексию**.

- **Онбординг**: 4-5 поворотных вопросов, не форма. Пример:
  - «Расскажи кратко в чём ты сейчас и что хочется изменить»
  - «Какие 2-3 области важны?»
  - «Что у тебя из ресурсов / ограничений (время, деньги, обязательства)?»
  - «Через 3 месяца как поймёшь что движение есть?»
- **Еженедельная рефлексия**: открывается раз в неделю кнопкой в Self.
  - Чат подтягивает базу знаний + последние выполненные задачи + рефлексии.
  - Спрашивает «что ты заметил», «куда сместить фокус», «надо ли заменить ось».
  - Может предложить: убрать ось, ввести новую, поменять вес оси, перегенерить план.

Чат — короткий (5-7 реплик), всегда опционален.

### 5. Фильтры и сортировка задач

На вкладке «Задачи» добавить:

- Фильтры: статус (active/done), ось (multi-select), дедлайн (today/this week/overdue/no due), источник (manual/from-roadmap), поиск по тексту.
- Сортировка: по дедлайну, по XP, по сложности, по дате создания.
- Сохранение последнего выбранного фильтра в SharedPreferences.

### 6. Визуал «графа осей» (бывшая пентаграмма)

#### Переименование

Внутреннее имя в коде `lifeAxes` остаётся. UI-имя — **«Древо»** (выбрано пользователем).

Метафора подходит:
- ось = ветвь;
- уровень = годовое кольцо / новый виток роста;
- закрытая задача = листок;
- levelUp = новая ветвь / прорастание.

#### Адаптивный многоугольник

- N осей → правильный N-угольник:
  - 3 → треугольник
  - 4 → квадрат
  - 5 → пентагон (как сейчас)
  - 6 → гексагон
  - 7..9 → соответствующие N-гоны
- При <3 осях рисуется вертикальный «термометр» на каждую.
- При >9 — переходим в радиальный bar-chart.

#### Уровни как кольца

- Каждый достигнутый уровень оси — внешний виток.
- Уровень 1: только внутренний полигон.
- Уровень 5: пять вложенных полигонов.
- Текущий progress в уровне — заливка от центра.

#### Интерактив

- **Тап по углу** (вершине оси) → раскрывается панель с историей выполненных задач этой оси.
- **Hover (десктоп)** → tooltip «ось N, уровень X, прогресс Y%».
- При **закрытии задачи**: короткая вспышка по затронутым осям (вес-пропорциональная).
- При **levelUp**: анимация прорастания нового кольца (1.5 сек) + конфетти-частицы.

#### Контракт компонента

```dart
class AxesGraph extends StatelessWidget {
  final List<LifeAxis> axes;
  final List<AxisScore> scores; // содержит level + progress
  final void Function(LifeAxis)? onAxisTap;
  final Animation<double>? burstAnimation; // при закрытии задачи
  ...
}
```

### 7. Персональная база знаний

`PersonalKnowledge` — отдельная таблица + JSON-документ:

```json
{
  "userId": "...",
  "summary": "Хочет стать Flutter QA. Уровень: уверенно. Работает full-time.",
  "goals": ["Получить оффер QA в течение 4 мес", "Подкачаться"],
  "constraints": ["1-2 ч в день в будни", "выходные свободны"],
  "preferences": {
    "learningStyle": "doing > reading",
    "avoidsBoringTutorials": true
  },
  "recentReflections": [
    { "date": "2026-04-20", "text": "Тяжело идут unit-тесты, нравится UI" }
  ],
  "completedHighlights": [
    "Прошёл flutter_test основы — easy",
    "Написал интеграционный тест на форму — hard, 3ч вместо 1"
  ]
}
```

Используется как **system-prompt addendum** при генерации задач:

```
[system prompt]
...основной промпт...

[user knowledge]
{summary}

Цели: {goals}
Ограничения: {constraints}
Что делал недавно: {recentReflections}
Что получилось: {completedHighlights}

Сейчас сгенерируй следующую задачу для оси {axisId}.
```

База знаний обновляется при:
- Заполнении опросника / чата онбординга.
- Сабмите Task Reflection.
- Еженедельной рефлексии.

Хранится локально + синхронизируется как остальные данные.

### 8. Оптимизация токенов

Несколько уровней:

#### a. Структурированный JSON output

Везде где LLM возвращает данные — `response_format: json_schema`. Никаких длинных описаний в свободной форме. Экономия 30-50% токенов.

#### b. Раздельные промпты по сложности

- **Цена×качество низкая** (`gpt-4o-mini` или дешёвый аналог): расчёт XP, простые подзадачи.
- **Цена×качество высокая** (`gpt-4o` / основная): генерация роадмапа, рефлексия.

#### c. Суммаризация и LRU

База знаний хранит **summary**, не полный лог. Раз в неделю:
- LLM сжимает последние N reflections в 1-2 предложения.
- Старые reflections архивируются (не идут в контекст).

#### d. Кэш на хеш запроса

Если пользователь нажимает «перегенерить роадмап» с теми же параметрами в течение 24 ч — отдаём из кэша. Хеш = SHA256(промпт + относящиеся axes/profile).

#### e. Локальный мини-расчёт

Расчёт XP, статистики, гейминга — **не вызывает LLM вообще**. Только генерация задач, рефлексия, levelUp-комментарий.

### 9. Что в приоритете (порядок реализации)

1. **Task Reflection окно + базовая БД пользователя**
   — без этого всё остальное бесполезно. 1-2 дня.
2. **`axisWeights` и детерминированный XP-расчёт**
   — большой рефактор LLM-промптов и repository.dart. 2 дня.
3. **Уровни осей + анимация levelUp**
   — добавить `level` в `AxisScore`, переписать пентагон-painter с поддержкой колец. 2 дня.
4. **Фильтры/сортировка задач** — 1 день.
5. **Чат-онбординг (опционально, скипуемое улучшение)** — 1-2 дня.
6. **Еженедельная рефлексия** — 1 день, после п.5.
7. **Кэш + структурированный JSON output** — 1 день, можно делать параллельно с п.2.
8. **Переименование пентагона + N-угольник адаптация** — 0.5 дня, чисто визуальное.

Итого ~10-12 дней работы. Можно дробить по PR (каждый пункт = отдельный PR).

### 10. Что **не** делаем в этой переработке

- Не вводим монетизацию / премиум-фичи.
- Не меняем синхронизационный протокол с бэкендом.
- Не отказываемся от локального хранения SQLite.
- Не добавляем мобильные платформенные фичи (виджеты, шорткаты) — отдельная история.

## Решения (из обсуждения)

- **Имя для пентаграммы** → **Древо**.
- **Длина онбординг-чата** → **7 реплик** (3-4 фактических вопроса + уточнения + резюме).
- **Когда показывать Task Reflection** → **по длительности задачи**. Конкретный порог:
  - `estimatedMinutes >= 30` или `actualMinutes >= 30` → показать.
  - короче — закрывается без окна, базовый XP.
- **Пороги уровней** → **возрастающие**:
  - Уровень 2: 100 XP
  - Уровень 3: 250 XP (т.е. +150 от прошлого)
  - Уровень 4: 450 XP (+200)
  - Уровень 5: 700 XP (+250)
  - Уровень 6: 1000 XP (+300)
  - Дальше: `next = prev + (50 + level * 50)` — каждый виток на 50 XP длиннее.

## Открытые вопросы (мелочь)

- **Что делать с уже накопленными данными при миграции** (текущие AxisScore без `level`)? Предложение: установить всем `level = 1`, `progress = текущий score`. Если score > 100 — каскадно повышать уровень.
